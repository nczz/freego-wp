# Security Policy

Please report security issues privately instead of opening a public issue.

Send a concise report with reproduction steps, affected version, and impact to the repository owner through GitHub.

## Scope

Security-sensitive areas include:

- WordPress admin actions and nonces
- stored issue data and rendered snippets
- GitHub update checks
- output-buffer HTML rewriting
- JavaScript runtime repair

Do not include real secrets, production cookies, access tokens, or private site content in reports.
